1. Информация о проекте

Сайт «ОператорПлюс» представляет информационный веб-ресурс об ИИ-решении для упрощения работы операторов контакт-центров Альфа-Банка. Продукт предлагает умного помощника, который обрабатывает типовые запросы, подсказывает ответы и снижает нагрузку на операторов без скриптов и лишних кликов.

Решение ориентировано на внедрение в контакт-центрах финансового сектора с учётом регламентов и баз знаний банка. Сайт информирует о возможностях системы, этапах внедрения, преимуществах и предоставляет контактные данные для связи.

2. Цели веб-сайта

  • Предоставить информацию о продукте «ОператорПлюс» и его возможностях для контакт-центров Альфа-Банка.
  • Обеспечить прозрачное представление этапов внедрения (аудит, настройка, пилот, масштабирование).
  • Поддержать имидж решения и привлечь заинтересованных заказчиков (заявки, демо).
  • Обеспечить круглосуточный доступ к информации о продукте и контактах.

3. Задачи веб-сайта

  • Предоставить информацию о компании и продукте (возможности, кейсы, «О нас»).
  • Описывать этапы работы и преимущества для Альфа-Банка.
  • Предоставить контактную информацию (почта, телефон) и призыв к действию (заявка, демо).
  • Обеспечить удобную навигацию по разделам без перезагрузки страницы (одна страница, переключение по якорям).

4. Целевая аудитория

  • Роли: руководители контакт-центров, менеджеры по внедрению, специалисты по качеству обслуживания, представители Альфа-Банка и других финансовых организаций.
  • География: Россия (в первую очередь).
  • Интересы: автоматизация операторов, ИИ в колл-центрах, соответствие регламентам, снижение нагрузки и времени обработки запросов.
  • Технические требования: современные браузеры (Chrome, Firefox, Safari, Edge), ОС Windows, macOS, Linux; адаптивное отображение на разных устройствах.

5. Логическая структура сайта

Сайт имеет одностраничную структуру с переключением разделов по якорям. Все разделы размещены в одном HTML-файле (index.html), видимым в каждый момент является один раздел (секция), соответствующий текущему якорю в адресе (#home, #features, #process, #about, #cases, #benefits, #contact).

5.1. Навигационная схема

  • Главная (#home) — герой-блок, ключевые метрики, кнопки «Узнать больше» и «Связаться с нами».
  • Возможности (#features) — три карточки: быстрые ответы, сокращение скриптов, меньше нагрузки.
  • Как работаем (#process) — три этапа: аудит и согласование, настройка и обучение, пилот и масштабирование.
  • О нас (#about) — миссия и технологии.
  • Кейсы (#cases) — кейсы и отзывы (пилот, обратная связь операторов).
  • Преимущества (#benefits) — список преимуществ и призыв «Оставить заявку».
  • Контакты (#contact) — текст приглашения, кнопка «Написать на почту», телефон.

В шапке и в подвале размещена одна и та же навигация (ссылки на якоря). Переход по ссылке меняет якорь в URL и отображает соответствующий раздел.

6. Макеты и структура страниц

Все «экраны» построены по единому принципу:

  • Шапка (header): фиксированная сверху; логотип «ОператорПлюс» слева, навигация справа; полупрозрачный фон с размытием (backdrop-filter), скруглённый блок.
  • Основная область: контейнер ограниченной ширины (max 1080px), контент по центру. Каждый раздел содержит: опциональную метку (section-label), заголовок (h2), вводный текст, затем карточки/списки/кнопки.
  • Подвал (footer): фиксированный снизу; логотип и слоган, дублирование навигации, копирайт и ссылка «Политика конфиденциальности».

Фон всей страницы — «космическое небо»: тёмный градиент с мягкими туманностями (фиолетовые/синие эллипсы) и звёздами (радиальные градиенты), без мелкой точечной сетки. Карточки и блоки имеют полупрозрачный тёмный фон, границы и тени для отделения от фона.

7. Средства реализации

  • Вёрстка: HTML5 (семантические теги: header, main, section, nav, footer, article). Один файл index.html.
  • Стили: один файл style.css. CSS-переменные для цветов, отступов, анимаций. Адаптив через медиазапросы (900px, 640px).
  • Шрифты: Manrope (Google Fonts), без засечек.
  • Инструменты: редактор кода (например, Cursor/VS Code), локальный сервер (например, Python http.server) для проверки.

8. Цветовая палитра и типографика

Фон: глубокий космос — #030308, градиенты до #0a0820, #080618; туманности — полупрозрачные фиолетовые и синие эллипсы.

Текст: основной #f2f4ff, вторичный #c4cce8, приглушённый #9ca4c4. Заголовки и акценты — градиенты (белый, розовый, фиолетовый).

Границы и поверхности: rgba(255,255,255,0.28) и 0.2 для линий; карточки — тёмные полупрозрачные блоки с границами и лёгкой внутренней тенью.

Акценты: красный #ef3124, розовый #ff4f8f, фиолетовый #7b4dff — кнопки, метки, градиенты.

Кнопки: основная — градиент красный–розовый–фиолетовый; вторичная — полупрозрачная с обводкой.

Размеры шрифтов заданы в rem и clamp для адаптивности; относительные единицы позволяют масштабирование в браузере.

9. Тестирование HTML-кода валидатором

Проверка разметки проводилась с помощью W3C Markup Validation Service (Nu Html Checker): https://validator.w3.org/nu/. Использовался режим «Validate by direct input»: содержимое файла index.html было отправлено на проверку (Content-Type: text/html; charset=utf-8).

9.1. Результат первой проверки (ошибки)

Валидатор выдал одну ошибку:

Тип: error
Строка 199, столбец 11–126
Сообщение: Bad value "mailto:operator-plus@example.com?subject=Запрос на демо для Альфа-Банка" for attribute "href" on element "a": Illegal character in query. Space is not allowed.
Фрагмент кода:
<a href="mailto:operator-plus@example.com?subject=Запрос на демо для Альфа-Банка" class="btn btn-primary btn-large">Написать на почту</a>

Причина: в значении атрибута href в ссылке mailto: в параметре subject= использовались пробелы. По стандарту в query-строке URL пробелы не допускаются — их необходимо закодировать (например, как %20).

9.2. Внесённые исправления

Ссылка «Написать на почту» была исправлена: пробелы в теме письма заменены на процентное кодирование пробела (%20).

Было:

href="mailto:operator-plus@example.com?subject=Запрос на демо для Альфа-Банка"

Стало:

href="mailto:operator-plus@example.com?subject=Запрос%20на%20демо%20для%20Альфа-Банка"

После исправления файл был повторно отправлен на проверку.

9.3. Результат повторной проверки

Проверка завершена успешно. Сообщений об ошибках и предупреждениях нет (messages: []). Документ соответствует выбранной схеме проверки. Язык документа определён как «ru» (атрибут lang="ru" на элементе html).

Итог: все выявленные ошибки устранены; HTML-разметка сайта проходит валидацию W3C без замечаний.
